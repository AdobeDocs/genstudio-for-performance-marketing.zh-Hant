# 波蘭文GenStudio發行說明

**標準目標檔案：** [help/user-guide/release-notes.md](../../../help/user-guide/release-notes.md)

**範例：** [範例.md](examples.md)

**起草工作流程（上游）：** [產生發行說明](../generate-release-notes/SKILL.md)

## 範圍（必要）

僅修飾&#x200B;**僅**&#x200B;使用者識別為&#x200B;**新加入的內容**&#x200B;此回合：

- **在檔案中：** `###`子區段在`help/user-guide/release-notes.md`中的單一標題`## YYYY.MM {#latest}`下。
- **不要**&#x200B;編輯&#x200B;**舊版發行說明** （可收合的`+++`區塊）下的頁面標題、簡介段落、前置內容或任何文字。
- **除非使用者**&#x200B;明確&#x200B;**要求，否則不要**&#x200B;編輯之前`##`個每月區塊或同一`{#latest}`區段中預先存在的`###`。
- 如果不清楚哪些`###`是新的，**詢問**&#x200B;或使用&#x200B;**git diff** /在變更文章之前編輯內容。
- **貼上內容：**&#x200B;使用者確認貼上符合那些&#x200B;**相同**&#x200B;新子區段時，僅會將Markdown修飾為貼上（不要將不相關的貼上視為範圍內）。

將差異保持在&#x200B;**最小**：僅文字和段落符號 — 不會重新調整檔案中其他位置的磁碟機。

## 語音和色調

- 針對產品發行說明： **conscentical**、**scannable**&#x200B;和&#x200B;**已啟動**&#x200B;關於&#x200B;**新值**，請撰寫為&#x200B;**撰稿人** — 以結果為導向，以及「可為您帶來什麼」，使用強動詞和清晰的權益語言。
- **保持準確：**&#x200B;除了功能之外，沒有其他宣告；符合Experience League / Adobe語調（自信且清晰，不是聳人聽聞的）。
- **利用**&#x200B;特定&#x200B;**權益與差異（行銷人員現在可以做什麼，可以移除哪些摩擦），而非空洞的炒作或未經證實的極致化，來產生興奮感**。

## 段落規則

- 每個段落： **2-3個句子**。 **一個段落中的句子不能超過三個** — 如果您需要更多，請開始一個&#x200B;**新的段落**。
- 選用性：如果段落仍以一般行長在演算的檔案中執行超過&#x200B;**~3行**，請進一步分割。

## 不要新增

- **程式「如何」**&#x200B;內容：編號步驟、「按一下&#x200B;**[!UICONTROL ...]** then...」、完整UI逐步解說或教學課程用語。 發行說明總結了&#x200B;**送出的內容**&#x200B;和&#x200B;**為什麼它很重要**，而不是動手實踐的課程。
- 違反產生技能上[禁止的內容](../generate-release-notes/SKILL.md#prohibited-content)的內容（無Jira金鑰、僅限內部的URL、wiki-as-proof等）。

## 保留（不要移除或重寫結構）

- `[!DNL …]`、`[!UICONTROL …]`、`[!BADGE …]`和其他ExL密碼。
- 現有&#x200B;**相對**&#x200B;檔案連結和錨點模式： `[phrase](/help/...)`在有意義的錨點文字上。
- Beta徽章區塊與[產生發行說明範例](../generate-release-notes/examples.md)中所使用的區塊完全相同。

## 工作流程檢查清單

1. [ ]確認`## … {#latest}`底下的&#x200B;**`###`在範圍中（此輪新增）**。
2. [ ]針對每個領域`###`，加強每個[語音和音調](#voice-and-tone)及[段落規則](#paragraph-rules)的復本。
3. [ ]移除或縮短任何&#x200B;**做法**&#x200B;指示；保留&#x200B;**使用者結果**。
4. [ ]確認連結和短程式碼仍然有效；根據[品質檢查](#quality-checks)執行內部ID或禁止模式的快速掃描。

## 品質檢查

- [ ]只有在`{#latest}`下同意的&#x200B;**新** `###`區塊已變更；封存和舊月份未變更。
- [ ]沒有新的Jira樣式ID、內部Wiki URL或「參閱票證」語言。
- [ ]段落是&#x200B;**2-3個句子**&#x200B;每個（每個段落最多三個句子）。
- [ ]副本保持&#x200B;**事實**&#x200B;並與描述的功能對齊。
